Module 1: Introduction to Networks

Hands-On Linux Checkpoint Assessment
🔬
Hands-On Lab — Network Basics
IP configuration & connectivity testing
01

Network Interface Identify பண்ணுங்க

Terminal open பண்ணி ip addr show run பண்ணுங்க. eth0 (wired) மற்றும் lo (loopback 127.0.0.1) interfaces காண்பிக்கும். உங்கள் IP address note பண்ணுங்க.

02

Subnet Mask புரிஞ்சுக்கோங்க

192.168.1.0/24 → /24 = 255.255.255.0. இந்த network-ல் 256 addresses இருக்கும் — 1 network, 1 broadcast, 254 usable hosts. /25 என்றால் 126 hosts, /26 என்றால் 62 hosts.

03

Ping Test பண்ணுங்க

Gateway-க்கு ping பண்ணுங்க: ping -c 4 192.168.1.1. 4 packets send → 4 receive ஆனால் = network OK. "Request timeout" வந்தால் = connectivity problem.

04

Traceroute — Path பார்க்க

traceroute google.com run பண்ணுங்க. உங்கள் packet எந்த routers-வழியாக travel பண்றது என்று காட்டும். ஒவ்வொரு line = ஒரு hop (router).

05

DNS Lookup

nslookup google.com அல்லது dig google.com run பண்ணுங்க. Domain name → IP address convert ஆவதை பாருங்க. DNS server 8.8.8.8 use பண்றால் Google DNS.

🐧
Linux Commands — Detailed Explanation
Every command with output explanation
💡ஒவ்வொரு command-ம் terminal-ல் type பண்ணி practice பண்ணுங்க. Output பார்த்து understand பண்ணுங்க. Questions வந்தால் note பண்ணிக்கோங்க.
# ═══ 1. NETWORK INTERFACES ═══
$ip addr show              # All interfaces list
$ip addr show eth0         # Specific interface
$ifconfig                  # Old method (net-tools)

# ═══ 2. CONNECTIVITY TEST ═══
$ping -c 4 8.8.8.8        # -c = count (4 packets)
$ping -c 4 -i 0.5 google.com # -i = interval (0.5 sec)
# Output example:
# 64 bytes from 8.8.8.8: icmp_seq=1 ttl=118 time=12.3 ms
# time= value குறைவாக இருந்தால் network fast

# ═══ 3. ROUTING TABLE ═══
$ip route show            # All routes
$route -n                 # Numeric format
# "default via 192.168.1.1" = default gateway

# ═══ 4. TRACEROUTE ═══
$traceroute google.com
$tracepath google.com     # Alternative (no sudo needed)

# ═══ 5. DNS LOOKUP ═══
$nslookup google.com
$dig google.com           # Detailed DNS info
$dig google.com +short   # IP only
$cat /etc/resolv.conf     # DNS server config file

# ═══ 6. OPEN PORTS ═══
$netstat -tuln            # -t=TCP -u=UDP -l=listening -n=numeric
$ss -tuln                # Modern replacement for netstat

# ═══ 7. HOSTNAME ═══
$hostname                 # Machine name
$hostname -I              # All IPs
$cat /etc/hosts           # Local DNS mappings

Bash Script — Network Monitor
5 different IPs-க்கு ping பண்ணி, UP/DOWN status print பண்ண bash script எழுதுங்க. Loop, if-else, exit code use பண்ணுங்க.
# Hard Task Solution — Network Monitor Script
#!/bin/bash

HOSTS=("8.8.8.8" "8.8.4.4" "192.168.1.1" "192.168.1.254" "10.0.0.1")

for host in "${HOSTS[@]}"; do
  ping -c 1 -W 2 $host > /dev/null 2>&1
  if [ $? -eq 0 ]; then
    echo "✅ $host — UP"
  else
    echo "❌ $host — DOWN"
  fi
done

# Run: chmod +x monitor.sh && ./monitor.sh

Linux Commands — EC2 Management
# ═══ SSH CONNECT ═══
$chmod 400 my-keypair.pem     # Permission: owner read-only
$ssh -i my-keypair.pem ec2-user@
# ec2-user = Amazon Linux default username
# ubuntu = Ubuntu AMI username

# ═══ SYSTEM INFO ═══
$uname -a                 # OS version info
$df -h                   # Disk usage (-h = human readable)
$free -h                 # RAM usage
$nproc                   # CPU core count
$top                     # Live process monitor (q to quit)

# ═══ EBS VOLUME MOUNT ═══
$lsblk                   # List block devices (attached volumes)
# xvda = root volume, xvdf = attached EBS
$sudo mkfs -t ext4 /dev/xvdf # Format as ext4
$sudo mkdir /data           # Mount point create
$sudo mount /dev/xvdf /data  # Mount volume
$df -h                   # Verify mount

# Auto-mount at reboot (/etc/fstab)
$sudo bash -c 'echo "/dev/xvdf /data ext4 defaults,nofail 0 2" >> /etc/fstab'

# ═══ WEB SERVER INSTALL ═══
$sudo yum update -y        # Update packages
$sudo yum install httpd -y # Apache install
$sudo systemctl start httpd # Start service
$sudo systemctl enable httpd# Auto-start at reboot
$sudo systemctl status httpd# Check status (Active: running ✓)

# Test webpage
$echo "<h1>Hello AWS!</h1>" | sudo tee /var/www/html/index.html
# Browser-ல் http://<PUBLIC-IP> type பண்ணுங்க

05

MFA Enable

Security → root account & IAM user-க்கு MFA enable பண்ணுங்க. Google Authenticator app → QR code scan → 6-digit code enter.

# ═══ CUSTOM IAM POLICY (JSON) ═══
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowS3ReadMyBucket",
      "Effect": "Allow",         // Allow or Deny
      "Action": [               // What actions allowed
        "s3:GetObject",        // Download file
        "s3:ListBucket"        // List files
      ],
      "Resource": [             // Which resources
        "arn:aws:s3:::my-bucket",
        "arn:aws:s3:::my-bucket/*"
      ]
    }
  ]
}

# ═══ AWS CLI — IAM Commands ═══
$aws iam list-users
$aws iam create-user --user-name dev-user
$aws iam list-attached-user-policies --user-name dev-user
$aws iam get-user --user-name dev-user
$aws iam list-groups
